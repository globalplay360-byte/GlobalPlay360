Resum Executiu

GlobalPlay360 en una sola pàgina

GlobalPlay360 és una plataforma web preparada per estructurar, ordenar i professionalitzar la relació entre esportistes, entrenadors i clubs. El projecte s'ha concebut com una base sòlida, escalable i orientada a negoci, amb una experiència d'ús clara i una infraestructura moderna capaç de suportar autenticació, dades en temps real, pagaments i creixement funcional.

Tipus de plataforma
Web App

Aplicació responsive pensada per desktop i mòbil.

Models d'usuari
3 rols

Player, Coach i Club amb experiència diferenciada.

Idiomes
3

Català, castellà i anglès integrats al producte.

Pagament
Stripe

Subscripcions, trial i portal de client integrats.

Objectiu del projecte

Crear un entorn digital on el talent esportiu pugui presentar-se amb criteri professional, on els clubs puguin publicar oportunitats i gestionar candidatures, i on la relació entre les parts es pugui convertir en un flux ordenat, traçable i monetitzable.

Enfocament de disseny

La plataforma s'ha desenvolupat amb una estètica premium, fosca i corporativa, prioritzant claredat, confiança i rapidesa d'ús. L'objectiu no ha estat només que funcioni, sinó que projecti una imatge de producte seriós, coherent i preparat per créixer.

GlobalPlay360 no és només una web informativa: és una base de producte real preparada per operar amb usuaris, dades, pagaments i fluxos de negoci.

Producte

Com està dissenyada la plataforma

El projecte s'ha estructurat perquè cada perfil tingui una experiència pròpia, però dins d'un mateix sistema coherent. Això permet que la plataforma sigui fàcil d'utilitzar avui i fàcil d'ampliar demà.

Perfil Player

Els esportistes poden crear un perfil professional, completar la seva informació esportiva, pujar avatar, veure oportunitats, enviar candidatures i comunicar-se amb clubs quan el model de negoci ho permet.

Perfil Coach

Els entrenadors disposen d'un perfil específic, amb els seus propis camps d'informació, i comparteixen la lògica de visibilitat, candidatures i accés a oportunitats segons el rol i el pla actiu.

Perfil Club

Els clubs poden editar el seu perfil, publicar ofertes, gestionar l'estat de les candidatures rebudes i iniciar converses amb els perfils seleccionats, centralitzant el flux de reclutament dins la mateixa plataforma.

Missatgeria i seguiment

El sistema incorpora xat en temps real, actualització d'estats de candidatura i indicadors de missatges no llegits, de manera que la plataforma no es limita a generar interès: acompanya el procés complet.

Clau de producte: la plataforma no es construeix només al voltant de pàgines, sinó al voltant de fluxos: registre, onboarding, perfil, oferta, candidatura, estat, conversa i subscripció.

Arquitectura

Com està construït tècnicament

L'arquitectura s'ha plantejat perquè el projecte tingui una base moderna, segura i mantenible. Això permet escalar funcionalitats sense reescriure el nucli del sistema.

Frontend

Interfície desenvolupada amb React i TypeScript, amb componentització reutilitzable, navegació per rutes i una capa visual coherent per a pàgines públiques, auth i dashboard privat.

Backend / BaaS

Firebase centralitza autenticació, base de dades, storage, cloud functions i sincronització. Això accelera desenvolupament i simplifica l'operativa sense renunciar a control tècnic.

Pagaments

Stripe gestiona el model de subscripció, el trial, el portal de client i la sincronització d'estat del pla, mantenint la lògica de billing alineada amb una infraestructura professional.

1

Autenticació

L'usuari es registra, verifica email i entra al flux segons el seu rol. La plataforma controla l'accés amb rutes protegides i estat autenticat.

2

Dades i lògica

Firestore gestiona perfils, oportunitats, candidatures, xats i missatges. Les regles de seguretat delimiten què pot llegir o modificar cada actor.

3

Billing i permisos

Stripe i Cloud Functions decideixen quin accés correspon a cada usuari, evitant inconsistències i centralitzant la veritat del sistema al nivell correcte.

Programari Utilitzat

Tecnologies i eines de treball

Per construir GlobalPlay360 s'ha utilitzat un stack actual i àmpliament adoptat, pensat per combinar rapidesa de desenvolupament, qualitat de codi i capacitat de manteniment.

Frontend

React + TypeScript + Vite

Base moderna per crear una aplicació ràpida, modular i tipada. Vite aporta velocitat de desenvolupament i build, mentre que TypeScript ajuda a reduir errors i augmentar la robustesa del codi.

Estil i experiència

Tailwind CSS + Framer Motion

Tailwind permet construir una interfície consistent i escalable; Framer Motion s'ha utilitzat per aportar transicions i microinteraccions amb una sensació més premium i cuidada.

Backend i dades

Firebase

Auth, Firestore, Storage, Hosting i Cloud Functions formen el nucli operatiu del projecte. Això permet tenir autenticació, dades en temps real, pujada de fitxers i lògica de servidor en un únic ecosistema.

Pagaments

Stripe

Gestió professional de plans, trials, canvi de subscripcions i customer portal. La integració s'ha pensat perquè el sistema de billing sigui fiable i alineat amb bones pràctiques.

Idiomes

i18next

El projecte inclou internacionalització real per permetre una operativa natural en català, castellà i anglès sense duplicar pantalles ni lògica.

Entorn professional

Git, GitHub i Visual Studio Code

Control de versions, treball per branques, commits semàntics i entorn de desenvolupament modern per garantir traçabilitat, ordre i evolució professional del projecte.

Seguretat i Fiabilitat

Criteris tècnics aplicats

Un dels objectius del projecte ha estat evitar solucions superficials. Per això s'han aplicat decisions que reforcen la consistència del producte i la confiança en el seu funcionament.

Regles d'accés

Les dades no depenen només del que permet la interfície. Firestore Rules reforcen què pot fer cada usuari per rol i per context.

Font de veritat del billing

El pla actiu no depèn d'un camp editat manualment, sinó de l'estat real sincronitzat amb Stripe, evitant inconsistències de permisos.

Base escalable

La separació de capes i la componentització permeten ampliar funcionalitats sense comprometre la base existent.

El resultat és una plataforma amb aparença de producte acabat, però també amb una base tècnica preparada per continuar evolucionant sense perdre ordre.

Valor del Projecte

Valor de futur

La infraestructura i el model de dades deixen oberta la porta a analítica, automatitzacions, nous plans, millores de matching o mòduls administratius més avançats.
